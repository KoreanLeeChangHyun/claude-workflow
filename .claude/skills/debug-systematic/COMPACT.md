# debug-systematic (Compact)

## 핵심 원칙
감으로 코드를 바꾸지 말고, 증거 기반으로 원인을 격리하여 수정한다.

## 4단계 근본 원인 분석 (RCA)
1. **증상 수집**: 에러 메시지, 스택 트레이스, 재현 조건, 발생 빈도, 최근 변경사항
2. **가설 수립**: 증상 기반 원인 후보 3개 이내 도출, 각 확률 추정
3. **격리 테스트**: 가설별 검증 실험 설계, 한 번에 하나만 변경하여 테스트
4. **수정 검증**: 근본 원인 확인 → 최소 변경 수정 → 재현 테스트로 수정 확인

## 수집 항목

| 항목 | 수집 방법 |
|------|----------|
| 에러 메시지 전문 | 로그, 콘솔, stderr |
| 스택 트레이스 | 전체 트레이스 보존 |
| 재현 조건 | 입력값, 환경, 타이밍 |
| 최근 변경사항 | git log, 배포 이력 |

## 금지
- 증상 수집 전 코드 수정
- 한 번에 여러 변수 변경
- 근본 원인 미확인 상태에서 "임시 수정" 적용
- 가설 없이 무작위 수정 시도
