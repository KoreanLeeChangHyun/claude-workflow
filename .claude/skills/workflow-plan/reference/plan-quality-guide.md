# 계획서 품질 가이드

## 완료 기준 작성 가이드

완료 기준은 Worker와 Reviewer가 완료 여부를 객관적으로 판정할 수 있도록 측정 가능하게 작성해야 한다.

**측정 가능한 완료 기준 3대 원칙:**

| 원칙 | 설명 | 적용 예시 |
|------|------|----------|
| 정량적 수치 포함 | "N개 이상", "N건", "N% 이상" 등 구체적 숫자 | "3가지 이상의 전략이 비교 분석됨" |
| 구체적 파일/경로명 포함 | 대상 파일을 명시하여 검증 범위 한정 | "`SKILL.md`에 체크리스트 섹션이 추가됨" |
| 검증 가능한 조건 | "~0건", "~이상", "~형식으로" 등 boolean 판정 가능 | "잔여 참조 0건", "`flowchart TD` 형식으로 변환됨" |

**주의**: "~됨", "~있음"만으로 끝나는 추상적 완료 기준은 지양한다. 반드시 위 3대 원칙 중 1개 이상을 적용한다.

## 워커별 작업 상세 세분화 규칙

워커별 작업 상세의 번호 항목은 균일한 세분화 수준을 유지해야 한다.

| 규칙 | 값 |
|------|-----|
| 권장 항목 수 | 4-5개 |
| 최소 항목 수 | 3개 |
| 최대 항목 수 | 7개 |
| 초과 허용 조건 | 대상 파일이 10개 이상인 특수 사유 시에만 |
| 워커 간 편차 | 동일 계획서 내 워커 간 항목 수 편차 2 이내 권장 |

**작성 원칙:**
- 각 항목은 구체적 행위 동사로 시작 (생성, 수정, 추가, 삭제, 분석, 검증 등)
- 한 항목에 복수 행위를 병합하지 않음 ("X를 수정하고 Y를 추가" -> (1) X 수정 (2) Y 추가로 분리)
- 모든 워커가 동일한 세분화 수준을 유지

## 계획서 품질 체크리스트

planner는 계획서 작성 완료 후 다음 체크리스트로 자가 검증한다. 항목을 만족하지 못하면 계획서를 보완한다.

- [ ] **비고 정량성**: 비고 섹션에 command에 맞는 정량적 데이터(수치, 크기, 개수 등)가 1개 이상 포함되어 있는가?
- [ ] **완료 기준 측정 가능성**: 각 완료 기준에 정량적 수치, 구체적 파일명, 또는 검증 가능한 조건이 포함되어 있는가?
- [ ] **워커 상세 균일성**: 모든 워커의 작업 항목 수가 3-7개 범위이고, 워커 간 편차가 2 이내인가?
- [ ] **비고 서브섹션 구조화**: 비고 내용이 ### 서브섹션으로 구분되어 있는가?
- [ ] **범위 명시성**: 작업 범위(또는 연구 범위)가 bullet 또는 테이블로 명확히 나열되어 있는가?
- [ ] **선택적 섹션 판단**: 불필요한 섹션을 형식적으로 채우지 않고 적절히 생략했는가? (아래 판단 기준 참조)
- [ ] **워커별 작업 상세 품질**: 각 워커의 작업 항목이 행위 동사로 시작하고, Worker가 추가 질문 없이 실행 가능한 구체성을 갖추었는가?
