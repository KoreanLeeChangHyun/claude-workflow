# 품질 레벨 프레임워크

명령어별로 기대되는 품질 수준을 명시적으로 정의합니다. Agentic Coding Spectrum (4단계)을 기반으로, 우리 시스템은 Level 1(Vibe Coding)을 허용하지 않으며 최소 Level 2 이상을 강제합니다.

## 품질 레벨 정의

| Level | 이름 | 테스트 요구수준 | 리뷰 깊이 | 검증 기준 |
|-------|------|---------------|----------|----------|
| 2 | Agentic Coding | 빌드 성공, 기본 동작 확인 | 기본 검증 (자동) | 빌드/실행 에러 0건 |
| 3 | Agentic SW Engineering | 단위 테스트 필수, 기존 테스트 통과 | 원샷 자동 리뷰 | 테스트 통과, 린트 클린 |
| 4 | High-Quality SW Engineering | 테스트 커버리지 분석, 엣지 케이스 | 심층 분석, 다중 소스 교차 검증 | 증거 기반 분석, 정량적 근거 |

## 명령어별 기본 품질 레벨

| 명령어 | 기본 Level | 테스트 요구수준 | 리뷰 깊이 |
|--------|-----------|---------------|----------|
| implement | Level 3 | 단위 테스트 필수 | 원샷 자동 리뷰 |
| refactor | Level 3 | 기존 테스트 통과 필수 | 변경 범위 검증 |
| build | Level 2 | 빌드 성공 | 기본 검증 |
| review | Level 4 | 테스트 커버리지 분석 | 인터랙티브 가능 |
| architect | Level 4 | 해당 없음 | 심층 리서치 기반 |
| analyze | Level 4 | 해당 없음 | 증거 기반 분석 |
| research | Level 4 | 해당 없음 | 다중 소스 교차 검증 |
| framework | Level 3 | 프레임워크 테스트 통과 | 원샷 자동 리뷰 |

## 계획서에서의 품질 레벨 활용

- 계획서 메타 정보에 품질 레벨을 명시하면, Worker가 해당 레벨의 검증 기준을 적용
- 명시하지 않으면 명령어별 기본 Level이 자동 적용
- 계획서에서 기본 Level을 override 가능 (예: 단순 implement를 Level 2로 낮추거나, 중요 build를 Level 3으로 높이기)

**계획서 메타 정보 예시:**
```markdown
- 작업 ID: 143000
- 명령어: implement
- 품질 레벨: Level 3 (기본값)
```

> **원칙**: 품질 레벨은 권장 가이드라인이며, 강제가 아닙니다. planner가 작업 특성에 따라 유연하게 조정할 수 있습니다.
