# research-general (Compact)

## 목적
기술/개념/도구에 대한 체계적 웹 조사 후 구조화된 리포트 제공.

## 워크플로우
1. 주제 파악 → 범위 정의
2. 정보 수집 (WebSearch + WebFetch 병행)
3. 교차 검증 (다중 소스 대조)
4. 분석/정리 → 리포트 작성

## 핵심 원칙
- 객관성: 편향 없이 사실 기반 정보
- 출처 명시: 모든 정보의 출처 URL 기록
- 교차 검증: 단일 소스 신뢰 금지, 최소 2개 소스 대조
- 실용성: 실제 적용 가능한 인사이트 도출

## 리포트 구조
1. 요약 (Executive Summary)
2. 배경/맥락
3. 핵심 발견
4. 비교 분석 (해당 시)
5. 권고사항
6. 출처 목록

## 금지
- 출처 없는 주장
- 단일 소스 의존
- 오래된 정보를 최신으로 제시
