# 출처 신뢰도 평가 기준

research 스킬 사용 시 수집된 정보 소스의 신뢰도를 평가하기 위한 가이드입니다.

## 소스 등급 체계

신뢰도 순으로 소스를 5등급으로 분류합니다. 상위 등급 소스를 우선 활용합니다.

| 등급 | 소스 유형 | 신뢰도 | 설명 |
|:----:|----------|:------:|------|
| S | 공식 문서, 표준 규격 | 최고 | 해당 기술/프로젝트의 공식 문서, RFC, W3C/ECMA 표준 등 |
| A | 학술 논문, 기술 보고서 | 높음 | 피어 리뷰된 논문, 기업 기술 블로그(공식), 벤치마크 보고서 |
| B | 전문가 기술 블로그 | 중간 | 검증된 전문가의 개인 블로그, 기술 매거진(InfoQ, ThoughtWorks 등) |
| C | 커뮤니티 콘텐츠 | 낮음 | Stack Overflow, Reddit, GitHub Discussions, 포럼 답변 |
| D | 비검증 콘텐츠 | 최저 | 출처 불명 블로그, AI 생성 의심 콘텐츠, 광고성 게시물 |

## URL 패턴별 기본 신뢰도 매핑

URL 패턴으로 소스의 초기 신뢰도 등급을 빠르게 판단합니다. 실제 내용 확인 후 등급을 조정할 수 있습니다.

| URL 패턴 | 기본 등급 | 예시 |
|----------|:--------:|------|
| `docs.*`, `*.dev/docs`, `developer.*` | S | docs.python.org, react.dev/docs, developer.mozilla.org |
| `*.gov`, `*.edu`, `*.ac.*` | S-A | 정부/교육기관 도메인 |
| `arxiv.org`, `doi.org`, `ieee.org`, `acm.org` | A | 학술 논문 저장소 |
| `engineering.*.com`, `blog.*.dev` (기업 공식) | A | engineering.fb.com, blog.rust-lang.org |
| `*.medium.com`, `dev.to`, `hashnode.dev` | B-C | 개인 기술 블로그 플랫폼 (저자에 따라 변동) |
| `stackoverflow.com`, `github.com/*/issues` | C | 커뮤니티 Q&A, 이슈 트래커 |
| `reddit.com`, `*.discourse.org` | C | 커뮤니티 포럼 |
| `w3schools.com` | C | 교육용이나 오류 빈번 |
| 나머지 | D | 개별 확인 필요 |

## 신뢰도 평가 절차

1. **URL 패턴으로 초기 등급 부여**: 위 매핑 테이블 참조
2. **내용 기반 등급 조정**: 아래 기준으로 등급을 상향 또는 하향 조정
3. **최종 등급 결정**: 리포트에 소스별 등급을 기록

### 등급 상향 요인

| 요인 | 설명 |
|------|------|
| 코드 예시 포함 | 실제 동작하는 코드 예시가 포함된 경우 |
| 날짜 최신 | 최근 6개월 이내 작성/업데이트 |
| 저자 전문성 | 해당 분야 인정받는 전문가 (커미터, 메인테이너 등) |
| 참조 문헌 포함 | 다른 신뢰할 수 있는 소스를 참조 |
| 버전 명시 | 기술의 특정 버전에 대해 명확히 기술 |

### 등급 하향 요인

| 요인 | 설명 |
|------|------|
| 날짜 오래됨 | 2년 이상 경과하여 정보 유효성 불확실 |
| 오류 발견 | 내용에 명백한 기술적 오류 포함 |
| 이해관계 편향 | 자사 제품 홍보 목적이 명확한 경우 |
| 출처 미표기 | 주장의 근거를 제시하지 않는 경우 |
| AI 생성 의심 | 전형적인 AI 생성 패턴 (반복적 구조, 모호한 일반론) |

## 리포트 적용 가이드

리포트의 "참고 자료" 섹션에서 소스별 등급을 함께 기록합니다.

```markdown
## 참고 자료

| 등급 | 출처 | 비고 |
|:----:|------|------|
| S | [React 공식 문서 - Hooks](https://react.dev/reference/react) | 공식 API 레퍼런스 |
| A | [React 18 Architecture (Meta Engineering)](https://engineering.fb.com/...) | Meta 공식 기술 블로그 |
| B | [Dan Abramov - useEffect 심층 분석](https://overreacted.io/...) | React 핵심 개발자 블로그 |
| C | [SO: React hooks best practices](https://stackoverflow.com/...) | 커뮤니티 (높은 투표 수) |
```

**핵심 결론은 S-A 등급 소스에 근거**: 리포트의 핵심 결론과 권장사항은 반드시 S 또는 A 등급 소스에 기반해야 합니다. B-D 등급 소스는 보조 정보로만 활용합니다.
