# 기본 명령어 템플릿

정보 제공, 분석, 설명 등 부작용이 없는 기본 명령어입니다.

## 템플릿

```yaml
---
name: <명령어-이름>
description: <무엇을 하는지 + 언제 사용할지>
---

# <명령어 제목>

## 수행 작업

<Claude가 따를 지침>

## 출력 형식

<출력 포맷 설명>
```

## 예시: 코드 설명 명령어

```yaml
---
name: explain-code
description: 코드를 다이어그램과 비유로 설명. "어떻게 동작해?", "이 코드 설명해줘" 등에 반응
---

코드 설명시 항상 포함:

1. **비유로 시작**: 일상적인 것에 비유
2. **다이어그램 그리기**: ASCII 아트로 흐름/구조 표현
3. **단계별 설명**: 무슨 일이 일어나는지 설명
4. **주의점**: 흔한 실수나 오해 지적

복잡한 개념은 여러 비유 사용. 설명은 대화체로.
```

## 예시: 보안 점검 명령어

```yaml
---
name: security-check
description: 코드의 보안 취약점 검사. "보안 점검해줘", "취약점 찾아줘" 등에 반응
allowed-tools: Read, Grep, Glob
---

OWASP Top 10 기준 보안 검사:

1. 인젝션 (SQL, Command, XSS)
2. 인증 결함
3. 민감 데이터 노출
4. XXE
5. 접근 제어 미흡
6. 보안 설정 오류
7. XSS
8. 역직렬화 취약점
9. 알려진 취약점 컴포넌트
10. 로깅/모니터링 미흡

## 출력 형식

| 심각도 | 위치 | 취약점 | 수정 제안 |
|--------|------|--------|-----------|
| High/Medium/Low | 파일:라인 | 설명 | 해결책 |
```

## 예시: 문서 생성 명령어

```yaml
---
name: generate-docs
description: 코드에서 문서 자동 생성. "문서 만들어줘", "README 작성해줘" 등에 반응
---

$ARGUMENTS에 대한 문서 생성:

1. 파일/함수 분석
2. 목적과 사용법 파악
3. 파라미터/반환값 문서화
4. 사용 예시 추가

## 문서 형식

```markdown
# {제목}

## 개요
{1-2문장 설명}

## 사용법
{코드 예시}

## API
{파라미터, 반환값}

## 예시
{실제 사용 예시}
```
