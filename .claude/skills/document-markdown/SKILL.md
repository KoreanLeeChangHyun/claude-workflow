# Markdown 작성 컨벤션

## 줄바꿈 규칙

| 상황 | 방법 |
|------|------|
| 단락 구분 | 빈 줄 1개 |
| 같은 블록 내 줄바꿈 | `<br>` 태그 |
| 표 셀 내 줄바꿈 | `<br>` 태그 |

**trailing double space 금지** — 의도가 보이지 않고 에디터/포매터가 제거할 수 있음. 항상 `<br>` 사용.

## 제목 구조

- `#` = 문서 제목 (파일당 1개)
- `##` = 주요 섹션
- `###` ~ `####` = 하위 섹션
- `#####` 이하는 사용 자제 — 깊으면 구조 재설계
- 제목 전후에 빈 줄 1개씩
- 제목에 인라인 코드 허용, 링크는 지양

## 표 (Table)

### 기본 문법

```markdown
| 헤더 A | 헤더 B | 헤더 C |
|--------|--------|--------|
| 값 1   | 값 2   | 값 3   |
```

### 정렬

```markdown
| 왼쪽 정렬 | 가운데 정렬 | 오른쪽 정렬 |
|:----------|:----------:|----------:|
| left      | center     | right     |
```

### 셀 내 줄바꿈

`<br>` 태그 사용:

```markdown
| 항목 | 설명 |
|------|------|
| A    | 첫째 줄<br>둘째 줄 |
```

### 셀 내 리스트

표 셀에서 불릿/넘버 리스트가 필요하면 `<br>` + 텍스트 마커로 표현:

```markdown
| 항목 | 세부 |
|------|------|
| 기능 | - 항목 1<br>- 항목 2<br>- 항목 3 |
```

### 병합 대안

GFM은 셀 병합을 지원하지 않음. 대안:
- 빈 셀에 위 값 반복 또는 `〃` 사용
- 복잡한 레이아웃은 HTML `<table>` 사용

## 리스트

- 비순서: `-` 통일 (`*`, `+` 혼용 금지)
- 순서: `1.` `2.` `3.` 실제 번호 부여
- 중첩: 2칸 또는 4칸 들여쓰기 (프로젝트 내 통일)
- 항목 간 빈 줄: 짧은 항목이면 없이, 긴 설명이면 빈 줄 1개

## 코드블록

- 인라인: 백틱 1개 `` `code` ``
- 블록: 백틱 3개 + 언어 태그

````markdown
```python
def hello():
    print("hello")
```
````

- 언어 태그 필수 (syntax highlight + 가독성)
- 블록 전후 빈 줄 1개

## 링크 / 이미지

```markdown
[표시 텍스트](URL)
[표시 텍스트](URL "툴팁")
![대체 텍스트](이미지경로)
```

- 상대 경로 우선 (프로젝트 내 문서)
- 긴 URL은 참조 링크 방식:

```markdown
자세한 내용은 [공식 문서][docs]를 참조.

[docs]: https://example.com/docs
```

## 인라인 HTML

GFM에서 허용되는 인라인 HTML 활용:

| 용도 | 태그 |
|------|------|
| 줄바꿈 | `<br>` |
| 접기/펼치기 | `<details><summary>` |
| 키보드 표기 | `<kbd>Ctrl</kbd>` |
| 상/하첨자 | `<sup>`, `<sub>` |
| 복잡한 표 | `<table>` |

`<div>`, `<span>` 등 레이아웃 태그는 최소한으로 사용.

## Frontmatter (YAML)

```yaml
---
name: value
description: >
  여러 줄 설명
---
```

- 파일 최상단에 위치
- `---`로 시작과 끝 구분
- 긴 값은 `>` (folded) 또는 `|` (literal) 블록 스칼라 사용

## 체크리스트

.md 파일 작성/수정 시 확인:

- [ ] `#` 제목이 파일당 1개인가
- [ ] 줄바꿈에 trailing space 대신 `<br>` 또는 빈 줄을 사용했는가
- [ ] 표 셀 내 줄바꿈에 `<br>`을 사용했는가
- [ ] 코드블록에 언어 태그가 있는가
- [ ] 리스트 마커가 통일되어 있는가 (`-` 단일 사용)
