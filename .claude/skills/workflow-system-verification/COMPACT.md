# workflow-system-verification (Compact)

## 핵심 원칙
증거가 먼저, 주장은 그 다음. 검증 증거 없이 완료를 주장하지 말 것.

## Gate Function (5단계)
1. 식별: 이 주장을 증명하는 명령어는 무엇인가?
2. 실행: 전체 명령어를 실행 (최신, 완전)
3. 확인: 전체 출력 읽기, exit code 확인, 실패 수 카운트
4. 검증: 출력이 주장을 확인하는가?
5. 그 다음에만: 주장을 수행

## 필수 검증 항목

| 시점 | 검증 명령어 |
|------|------------|
| 코드 변경 후 | 빌드/컴파일 확인 |
| 테스트 관련 | 전체 테스트 스위트 실행 |
| 린트/포맷 | lint + format 실행 |
| 커밋 전 | 모든 검증 통과 확인 |

## 점진적 검증
변경 → 즉시 빌드/테스트 → 실패 시 즉시 수정 (누적 금지)

## 금지
- 단계 건너뛰기 = 검증이 아닌 거짓
- 이전 실행 결과 재사용 (반드시 최신 실행)
- "통과할 것으로 예상" 류의 추측
