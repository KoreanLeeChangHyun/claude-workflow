# REVIEW.md

프로젝트별 코드 리뷰 설정 파일. `review-comprehensive` 스킬이 이 파일을 로드하여 confidence threshold, 파일 패턴 기반 Critical 승격, 리뷰 제외 패턴, 중점 영역 우선순위를 적용한다.

## confidence_threshold

이슈 신뢰도 필터 기준값. 이 값 미만의 confidence를 가진 이슈는 `filtered_issues`로 분류된다.

| 설정 | 값 | 설명 |
|------|-----|------|
| 기본값 | 80 | 0-100 범위. 낮출수록 더 많은 이슈가 포함됨 |
| 범위 | 0-100 | 0 = 모든 이슈 포함, 100 = 최고 신뢰도 이슈만 포함 |

이 값을 오버라이드하려면 아래 설정을 수정한다:

```yaml
confidence_threshold: 80
```

## always_check

이 패턴에 매칭되는 파일의 이슈는 confidence 점수와 무관하게 자동으로 Critical로 승격된다. 보안·인증·환경 설정 등 민감 영역을 포함한다.

| 패턴 | 설명 |
|------|------|
| `**/auth/**` | 인증/인가 관련 모듈 |
| `**/security/**` | 보안 관련 모듈 |
| `**/.env*` | 환경 변수 파일 |
| `**/secrets/**` | 비밀 키/자격증명 디렉터리 |
| `**/credentials/**` | 자격증명 파일 |
| `**/payment/**` | 결제 처리 모듈 |
| `**/admin/**` | 관리자 전용 기능 |

패턴 추가/제거는 이 테이블을 직접 수정한다. glob 문법을 사용한다 (`**` = 임의 경로, `*` = 임의 파일명).

## ignore_patterns

리뷰에서 제외할 파일 및 디렉터리 패턴. 빌드 산출물, 외부 의존성, 압축 파일 등 리뷰 불필요 대상을 정의한다.

| 패턴 | 설명 |
|------|------|
| `node_modules/` | npm 패키지 의존성 |
| `dist/` | 빌드 산출물 |
| `build/` | 빌드 산출물 |
| `*.min.js` | 압축된 JavaScript |
| `*.min.css` | 압축된 CSS |
| `coverage/` | 테스트 커버리지 리포트 |
| `*.lock` | 패키지 락 파일 |
| `.workflow/` | 워크플로우 작업 디렉터리 |

## review_focus

프로젝트 특성에 맞게 리뷰 영역별 우선순위를 커스터마이즈한다. 높은 숫자일수록 높은 우선순위이며, 판정 기준의 가중치 계산에 반영된다.

| 영역 | 우선순위 | 설명 |
|------|---------|------|
| security | 1 | 최우선 - 보안 취약점 탐지 |
| code_quality | 2 | 코드 품질 및 유지보수성 |
| architecture | 3 | 아키텍처 일관성 |
| performance | 4 | 성능 최적화 |
| testing | 5 | 테스트 커버리지 |
| requirements | 6 | 요구사항 충족도 |

우선순위는 1(최고)부터 6(최저)까지 중복 없이 설정한다. 프로젝트 특성에 따라 순위를 재배치한다.
