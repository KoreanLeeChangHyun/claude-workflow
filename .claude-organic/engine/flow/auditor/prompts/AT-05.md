# AT-05: Context Sufficiency

## Evaluation Dimension

Assess whether the `context` field provides enough background for a worker
to understand the task without external knowledge — specifically whether it
includes prior tickets/commits, user intent, previous failure reasons, and
relevant memory references.

## Scoring Criteria

| Score | Verdict | Criterion |
|-------|---------|-----------|
| 5     | PASS    | ≥ 3 of 4 context elements present: (1) prior ticket/commit reference, (2) user intent quote or paraphrase, (3) previous failure or constraint reason, (4) memory/canon reference |
| 4     | PASS    | 2 of 4 context elements present and well-articulated |
| 3     | WARN    | 1 context element present, or 2 elements present but very brief (< 50 chars total) |
| 2     | FAIL    | Context has ≤ 1 item with no meaningful background beyond restating the goal |
| 1     | FAIL    | Context field is absent or empty |

## Bias Mitigation

Order of the 4 context elements is irrelevant; presence counts regardless of
position or labeling.
