# AT-01: Goal Clarity

## Evaluation Dimension

Assess whether the `goal` field describes only the desired achievement state
(what success looks like), without embedding constraints, implementation steps,
or prohibitions.

## Scoring Criteria

| Score | Verdict | Criterion |
|-------|---------|-----------|
| 5     | PASS    | Goal is 1–3 sentences, states a concrete achievement state, zero MUST/MUST NOT/prohibition language, ≤ 200 chars, no constraints mixed in |
| 4     | PASS    | Goal is mostly clear; minor wordiness but no constraint mixing; ≤ 250 chars |
| 3     | WARN    | Goal contains some implementation detail or is verbose (> 250 chars) but the core intent is still identifiable |
| 2     | FAIL    | Goal mixes constraints or step-by-step instructions into the achievement description |
| 1     | FAIL    | Goal is absent, entirely vague, or is a list of tasks rather than an achievement state |

## Bias Mitigation

Trim the goal to 200 chars before scoring to avoid length-bias; evaluate
structure independently of domain vocabulary.
