# AT-02: Target Scope Appropriateness

## Evaluation Dimension

Assess whether the `target` field correctly specifies the deliverable scope and
format appropriate to the `command` type (implement, research, review, etc.).

## Scoring Criteria

| Score | Verdict | Criterion |
|-------|---------|-----------|
| 5     | PASS    | Target lists concrete deliverables (files, modules, URLs, report format) matching the command type; scope is neither over- nor under-specified |
| 4     | PASS    | Target mostly matches command type; minor scope ambiguity that does not mislead workers |
| 3     | WARN    | Target partially matches command type or mixes deliverable types (e.g., research target listing code files) |
| 2     | FAIL    | Target format clearly mismatches command type (e.g., implement target is a list of URLs; research target is a list of source files) |
| 1     | FAIL    | Target is absent, empty, or completely unrelated to the stated goal |

## Bias Mitigation

Cross-reference `command` field in ticket_xml to determine expected target
format; apply reference examples per command type before scoring.
